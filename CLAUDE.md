# CLAUDE.md

## What this repo is
Concierge (working name — TBD, see Open TBDs) is an AI-native, WhatsApp-first
wedding-tech product aimed at breaking iPlan's ~70% market grip in the Israeli
wedding market. The product strategy — market analysis, why iPlan is beatable,
the wedge (win the RSVP decision via a free, better mobile seating chart + a
"Venue View" export that lets couples decline iPlan), v0 scope, anti-scope,
and kill criteria — is the **constitution**: `docs/iPlan-Competitor-Strategy.md`.
Don't relitigate product strategy here; if it conflicts with this file, the
constitution wins for product scope, this file wins for repo conventions.

## Operating model
This repo is driven by an autonomous dev flywheel: Ideation → Prioritization →
Development → QA/CR, orbiting Linear as the state machine. The shared
operating manual every one of those four agents reads first is `FLYWHEEL.md`
at repo root. Full detail on the loop and the agent prompts:
`docs/Autonomous-Dev-Flywheel.md` and `docs/FlywheelAgentPrompts.md`.

As of this session, the four agents are **not wired yet** — this is
scaffolding only. `AGENTS.md` at repo root is a separate, unrelated file
auto-generated by `create-next-app` with framework-version warnings.

## Kill switch
Any autonomous agent must stop immediately if the Linear issue labeled
`FLYWHEEL:PAUSED` exists in team Concierge, OR the repo file
`.flywheel/PAUSED` exists. That file does not exist by default. See
`FLYWHEEL.md` for the full protocol.

## Stack (locked)
Per `docs/Autonomous-Dev-Flywheel.md` §10.2 — do not deviate without updating
that doc first:
- **Full-stack:** Next.js (TypeScript, App Router) on Vercel
- **Backend/data:** Supabase (Postgres + auth + storage + edge functions)
- **AI:** Anthropic API (future — the concierge)
- **WhatsApp:** WhatsApp Business API (future)
- **Payments:** PayPlus/Cardcom (future, always-human 🔒, see below)
- **Tests/CI/deploy:** Vitest (unit) + Playwright (e2e/smoke) · GitHub Actions
  · Vercel preview→prod · Supabase branch for staging DB (deploy pipeline not
  wired yet, see CI/CD below)

## Repo conventions
1 product = 1 Linear team = 1 repo = 1 `CLAUDE.md`.

| Folder | Purpose |
|---|---|
| `app/` | Next.js App Router pages/routes |
| `lib/` | Shared logic, not UI |
| `components/` | Shared React components |
| `supabase/` | Supabase config + migrations |
| `tests/unit/` | Vitest unit tests |
| `tests/e2e/` | Playwright e2e/smoke tests |
| `docs/` | Strategy + process docs |
| `.flywheel/` | Autonomous-loop kill-switch file lives here |

Don't create feature folders (e.g. `app/rsvp`, `lib/seating`) before a real
ticket needs them — no scaffolding ahead of scope.

## 🔒 Boundaries — never autonomous
payments / gifting · authentication · DB migrations · anything touching guest
PII · WhatsApp message-template changes. See `FLYWHEEL.md` for the full
escalation protocol.

## Testing
- `npm test` — Vitest unit tests (`tests/unit/`)
- `npm run test:e2e` — Playwright e2e/smoke tests (`tests/e2e/`); run
  `npm run test:e2e:install` once to fetch browsers
- CI gate: lint + typecheck + test + build must all pass before merge (see
  CI/CD below)

## CI/CD
GitHub Actions (`.github/workflows/ci.yml`) runs `lint`, `typecheck`, `test`,
`build` on every PR and on push to `main`. Playwright e2e is intentionally
**not** in CI yet — it needs `playwright install --with-deps` plus a running
app instance, more machinery than this stage needs. Add it once a real
page/flow exists worth smoke-testing.

The deploy pipeline (Vercel preview→prod, Supabase staging branch, canary,
auto-rollback) is not wired yet — future work, see
`docs/Autonomous-Dev-Flywheel.md` §4/§9 (Layer 1b).

Branch protection on `main` must be configured manually — see
`docs/branch-protection-checklist.md`.

## Environment & secrets
No real values are live yet. `.env.example` lists every var anticipated so
far (Supabase, Anthropic, WhatsApp, PayPlus/Cardcom). Real values live in
Vercel/Supabase dashboards, never committed.

## v0 scope (forward context only — not built yet)
Per `docs/iPlan-Competitor-Strategy.md` §14: guest list (Excel + CSV import),
WhatsApp-native multilingual RSVP, mobile-first seating with AI auto-seating,
free read-only "Venue View" export, flat transparent pricing, instrumentation
+ a viral hook. Anti-scope: venue/vendor SaaS, full AI concierge, vendor
marketplace, payments/gifting. Don't scaffold folders for any of this until a
ticket lands.
